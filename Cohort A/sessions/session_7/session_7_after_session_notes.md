# Session 7 After-Session Notes: Add AI Prep Plan Agent

## What We Built Today

Today we added the AI Prep Plan Agent to the AI Interview Prep Copilot.

The feature takes:

- Target Role
- Weak Topics (pre-filled from saved profile)
- Days Until Interview
- Daily Available Hours

And generates:

- A personalized 7-day interview preparation plan
- Day-wise topics for each of the 7 days
- 3 to 5 practice questions per day
- 1 mock interview task per day
- A final revision checklist at the end of the plan

The plan is displayed inside the app as collapsible day cards.

---

# Why This Feature Matters

Most students preparing for interviews have limited time and multiple weak areas.

Without a plan, students:

- study randomly without structure
- spend too much time on comfortable topics
- ignore critical weak areas
- feel overwhelmed and do not know where to start

The AI Prep Plan Agent solves this by using the student's actual profile — their role, their weak areas, their available time — and producing a structured, realistic daily roadmap.

This is also one of the most interview-relevant features in the entire app because it demonstrates real agent behavior: AI taking goals and constraints and producing a structured multi-step action plan. This is a concept that comes up frequently in AI product and engineering interviews.

# What is an AI Agent?

An AI agent is a system that receives a goal and a set of constraints, and uses them to produce a structured multi-step plan or set of actions.

In plain language: imagine asking a friend who is an expert coach. You tell them: "I want to prepare for a software engineering interview in 7 days. I am weak in SQL and system design. I can study for 2 hours each day." A helpful coach does not just say "study SQL." They say: "Here is what you do on Day 1, Day 2, Day 3 — each day focused, each day realistic given your time, each day building toward your goal." That is agent behavior.

A chatbot, by contrast, responds to one message at a time. If you ask a chatbot "What should I study for a software interview?", it gives you a list. It does not know your weak areas. It does not know your time constraint. It does not plan across 7 days for you. It responds. It does not plan.

The difference is this: chatbots respond. Agents plan.

Our Prep Plan Agent is agent-like because it takes goals (target role, weak topics) and constraints (days available, hours per day) and produces a structured, multi-day, actionable output. The AI is not just answering a question — it is acting as a planner.

# What Happens When the Student Clicks Generate Plan?

```text
When the student clicks Generate Plan, the app first checks that all required inputs are filled — days until interview and daily hours must be provided. If validation passes, the app reads the saved profile from localStorage to get additional context like skills and project details. It then constructs a structured prompt that includes: the target role, weak topics, days available, and hours per day. This prompt tells the AI to return the plan as structured JSON with 7 day objects, each containing a topic, practice questions, and a mock task, plus a revision checklist. The prompt is sent to the AI. While the AI is processing, the app shows a loading message. When the response arrives, the app parses the JSON and renders 7 collapsible day cards. If the AI call fails, the app shows an error message asking the student to try again.
```

# Common Issues and Fixes

## Issue 1: Day cards are not rendering — plan appears as plain text

Possible reason:

- AI returned plain text instead of structured JSON
- Response parsing code is not handling the JSON format correctly
- The prompt did not explicitly ask for JSON output

What to ask AI:

```text
The AI Prep Plan Agent is returning the plan as plain text instead of structured day cards. Please update the AI prompt to explicitly ask for JSON output in this format: an object with a "plan" array of 7 day objects (each with day, topic, questions array, and mockTask) and a "revisionChecklist" array. Then update the response parsing code to extract and use this JSON. Show me the updated prompt string and the updated parsing logic.
```

---

## Issue 2: Target role and weak topics are not pre-filling in the form

- localStorage key name does not match what was saved in Session 1

- The form inputs are not being set after localStorage is read

# Key Takeaways

1. **Agent vs chatbot is the most important concept from Session 7**
   A chatbot responds to one message. An agent takes goals and constraints and produces a structured multi-step plan. Our Prep Plan Agent is agent-like because it uses role, weak topics, days, and hours to generate a 7-day roadmap — not just a single reply.

2. **Structured prompts produce structured outputs**
   The quality of the generated plan depends entirely on the quality of the prompt. A vague prompt produces a generic plan. A prompt that includes role, weak areas, time constraint, and an explicit JSON output format produces a personalized and renderable plan.

3. **Pre-filling inputs from saved profile makes the app feel connected**
   When the Prep Plan form automatically loads the student's target role and weak topics from localStorage, the app feels like one product — not 7 separate features. This kind of state reuse across features is an important product thinking signal in interviews.

4. **UI thinking matters alongside AI thinking**
   Collapsible day cards, a revision checklist, loading states, and validation are not AI features — they are product features. Students who can explain both the AI logic and the UX decisions behind this feature demonstrate a well-rounded understanding of product building.
