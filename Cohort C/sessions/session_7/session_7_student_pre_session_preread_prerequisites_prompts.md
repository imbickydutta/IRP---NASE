## Key Concepts to Revise Before This Session

### 1. What is a Vision-Language Model (VLM)?

A Vision-Language Model is an AI model that can accept both images and text as input and generate text as output. Gemini 1.5 Flash is a VLM. The same `genai.GenerativeModel("gemini-1.5-flash")` you have been calling for text generation can also receive an image alongside a text prompt.

### 2. How does Gemini handle image input in the `google-generativeai` library?

The `model.generate_content()` method accepts a list of content parts. For multimodal input, you pass a list containing both a text string and a PIL Image object:
```python
import google.generativeai as genai
from PIL import Image

model = genai.GenerativeModel("gemini-1.5-flash")
image = Image.open("sample_image.png")
response = model.generate_content(["Extract text from this image.", image])
```

### 3. What is PIL / Pillow?

Pillow is a Python library for image processing. `PIL.Image.open(path)` opens an image file and returns an Image object that Gemini can accept. You do not need to know image processing deeply — you only need `Image.open()` for this module.

### 4. What is traditional OCR and how is VLM-based OCR different?

Traditional OCR tools like pytesseract use computer vision algorithms to detect character-shaped regions in a binarized image and match them to characters. They are fast, deterministic, and free. VLM-based OCR uses a language model to "read" and interpret the image contextually — it understands document structure, can infer missing labels, and handles varied layouts. The trade-off is that VLMs are slower, cost API credits, and can hallucinate (produce plausible-sounding but incorrect values).

### 5. What is a confidence note in AI output?

A confidence note is a field in the output where the AI model documents its own uncertainty about a specific extraction. For example: "The date field appears partially obscured; extracted value may be incorrect." This is not a bug fix — it is a design feature. In production, non-empty `confidence_notes` triggers human review rather than automatic processing.

### 6. How does structured JSON output work in this context?

This is the same concept from Session 1's `structured_output_engine.py`. You tell Gemini exactly what JSON structure to return in the prompt, including the exact key names. Gemini follows the instruction and returns a JSON string in its response text. You then parse it with `json.loads()`. The new challenge here is that Gemini sometimes wraps the JSON in markdown code fences (` ```json ... ``` `), which need to be stripped before parsing.

### 7. Why is `os.environ.get("GEMINI_API_KEY")` better than hardcoding the key?

Hardcoding an API key in a Python script means the key is exposed whenever the file is shared, committed to Git, or copied. Using environment variables keeps the key outside the code. Always load it with `os.environ.get("GEMINI_API_KEY")` or using the `python-dotenv` library.

### 8. What is the `if __name__ == "__main__"` pattern and why does it matter here?

This pattern makes a Python script both runnable directly and importable as a module. When you run `python vision_ocr_module.py`, the `main()` block executes. When another script does `from vision_ocr_module import analyze_image`, the `main()` block does not execute — only the `analyze_image` function is available. This matters because a future agent or pipeline might want to call `analyze_image()` without running the full script.
